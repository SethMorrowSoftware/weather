Inputs (sources)

Add the signals your rules can use. Each one becomes a metric the rule builder discovers automatically. In the live app these are saved to config.yaml and validated before writing; new metrics apply on the next poll (mqtt/http connections re-read at the next cycle, a new topic subscription needs a service restart).

Operator variables

Virtual flags/setpoints you toggle from the dashboard (metric var_<name>). Type bool or number.

MQTT sensor inputs

Subscribe to another device's topic and expose its payload as a metric (number/bool/string).

HTTP JSON inputs

Poll a JSON endpoint and map fields (dotted path, arrays by index) to metrics.

Computed metrics

Derive a new number metric from others with a small formula: + - * / // % ** and parentheses over any metric defined above it. E.g. power_kw - solar_kw. A missing input yields no value (rules hold, fail-safe).
