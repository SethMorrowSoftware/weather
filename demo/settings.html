Settings

Configure the controller. Everything is validated before saving — out-of-range values are rejected and nothing is written. Thresholds, the lookback window, the poll interval, MQTT publish options and rules apply on the next poll; changing location, the MQTT connection, or any web setting needs a service restart. Passwords/tokens are never shown back — leave a field blank to keep the stored value.

Slack alerts

Get a Slack message if the MQTT broker stays unreachable. Needs a Slack bot token (xoxb-…) with chat:write, invited to the channel.

Remote status page

Push a read-only copy of the status to an external dashboard each cycle (see cloud-status/). Outbound only — nothing can reach back in, no remote control.

Web interface

Manual control lets an authenticated operator force a device ON/OFF from the dashboard (LAN-only, audited). Allow MQTT publishing lets the MQTT console send arbitrary messages to the broker. Both require a login; the remote status page stays read-only.

⚠ Changing location, the MQTT connection, or any web interface setting restarts the corresponding service in the live app. Thresholds, lookback, poll interval, QoS, retain, status topic and rules apply on the next poll automatically.

Metric history

Record each cycle's numeric metrics to a small local database so the History page can chart trends. Applies on the next poll.
